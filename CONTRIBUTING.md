# Contributing to FlowOS

Thanks for helping improve FlowOS.

## How to contribute

1. **Open an issue** first for larger changes (architecture, new integrations) so we can align.
2. **Fork** the repo and create a **focused branch** (`fix/…`, `feat/…`).
3. Keep PRs **small and reviewable** — one concern per PR when possible.
4. Run **`npm run lint`** and **`npx tsc --noEmit`** before pushing.
5. **Do not commit** `.env.local` or API keys.

## Code style

- Match existing patterns in the file you edit (imports, naming, formatting).
- Prefer clear names and minimal abstraction unless the change warrants it.

## Security

If you find a vulnerability, please report it privately to the maintainers rather than filing a public issue.
